엮음 (Yeoum) 디자인 핸드오프

폭주하는 생각을 마구 쏟으면, AI가 유의미한 걸 뽑아 '완성된 결과물'로 엮어주는 발산형(ADHD) 인재의 결과물 엔진. 이 문서는 개발 구현에 필요한 디자인 토큰·컴포넌트·규칙을 한 장에 정리한다.
사람 = 발산(강점) / AI = 수렴 보완 / 결과물 = 글·기획.

1. 설계 헌법 — R1~R10

모든 화면·컴포넌트가 지키는 타협 불가 규칙. ADHD 입력 피로 최소화가 목적.

R1
0탭 캡처 — 홈이 곧 입력창. "새로 만들기" 없음.
R2
음성 우선 — 큰 마이크가 지배 요소, 텍스트는 보조.
R3
입력 중 강제 0 — 제목·태그·저장버튼 금지. 자동저장만.
R4
미완성 허용 — 문법·완결 강요 UI 없음.
R5
언제든 중단 — "정말 나갈래?" 모달 금지.
R6
빈 화면 공포 방지 — 회전형 은은한 프롬프트.
R7
독촉 금지 — 빨간 배지·스트릭·연체 없음.
R8
한 화면 = 한 액션 — Primary 버튼 화면당 1개.
R9
작은 보상 — 부드러운 피드백. 게이미피케이션 X.
R10
감각 차분 — 넉넉한 여백, 저채도, 요소 최소.

6. 카피 · 보이스

짧고 다정하게, 비난·독촉 없이. 이모지 미사용. (토스 라이팅 8원칙을 엮음 톤으로 각색.)

이렇게
  • 뭐든 쏟아
  • 엮어줘
  • 받아쓰기가 잘 안 됐어요. 한 번 더?
  • 당신의 조각 12개 → 이 글이 됐어요
이렇게 말고

  • 음성 인식 오류 발생
  • 3일째 미작성! 스트릭이 끊겨요
  • 정말 나가시겠습니까?

금지 어휘: 마감 · 연체 · 놓침 · 벌점 · 스트릭 · "정말 나가시겠습니까". 숫자로 증폭 가시화: "조각 N개 → 결과물" Before/After 대비가 핵심 카피 장치.

7. 아이코노그래피

  • 얇은 스트로크(1.5~2px) 라인 아이콘. 색은 currentColor 상속. 채운(filled) 아이콘·이모지·유니코드 대용 금지.
  • 핵심 글리프: mic(쏟기) · type(입력) · git-merge(엮기) · copy·share(내보내기) · x(빼기) · check(선택).
  • 대체 플래그: 소스에 아이콘 세트 없음 → Lucide CDN으로 임시 대체. 실제 세트 확보 시 교체.
  • 로고 없음 — 마크 자리엔 Pretendard 600 워드마크 "엮음"으로 조판. 로고를 그리거나 지어내지 않는다.

8. 경계선 — 넣지 않는 것

"쪼개되 사람의 할 일이 아니라 결과물을 쪼갠다 / 깊이는 기능 수가 아니라 변환의 질로."

  • 일정관리·리마인드·투두·목표→태스크 매니저 — 보호 프레임 영역, 정체성 붕괴.
  • 발산 지도·사고 그래프 — 옵시디언 짝퉁 자초 + "정리 도구"라 ADHD가 못 쓰는 바로 그것.
  • "뇌 개선/증상 완화/시냅스" 치료 주장 — 확장 이탈.

단, '기획'류 결과물 끝의 실행 3단계는 허용 — 이는 산출물의 내용일 뿐 사용자의 하루를 추적하지 않는다.

9. 참조

  • styles.css — 전역 진입점 (tokens/* @import). 소비 프로젝트는 이 파일 하나만 링크.
  • tokens/ — colors · typography · spacing · fonts(Pretendard CDN)
  • components/ — core · capture · fragments
  • ui_kits/yeoum-app/ — 인터랙티브 목업 (홈·지도 제거·보관함·설정 + 쏟기→엮기→결과물→되물음)
  • 전체 상세는 프로젝트 readme.md 참조.
